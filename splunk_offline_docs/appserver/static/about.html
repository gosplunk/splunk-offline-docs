Splunk Offline Docs

Browse help.splunk.com product documentation inside Splunk Enterprise — no internet required after install.

What this app does

Splunk Offline Docs mirrors on-premises Splunk product documentation from help.splunk.com into your Splunk instance. Users get searchable HTML topics, product navigation, version selection, and dark mode — similar to the online help experience.

Documentation products in the bundle

ProductSource on help.splunk.comNotes
Splunk Enterprise splunk-enterprise Latest 10.x line
Enterprise Security splunk-enterprise-security-8 ES 8 documentation tree
SOAR (on-premises) splunk-soar/soar-on-premises Cloud SOAR docs excluded
IT Service Intelligence splunk-it-service-intelligence ITSI on-prem docs
Not in the default bundle: Splunk Cloud-only products, Observability Cloud, UBA, Attack Analyzer, and other help.splunk.com trees not listed above. Add products in scraper/products.yaml and rebuild the bundle to include more.

App navigation

  • Splunk Offline Docs — documentation browser (all users with app access)
  • About — this page
  • Configuration — bundle metrics, manual updates, daily check toggle (admins only)

Install

  1. Extract the app into $SPLUNK_HOME/etc/apps/
  2. Set ownership: chown -R splunk:splunk .../splunk_offline_docs
  3. Restart Splunk
  4. Open the app from Splunk Web navigation
tar -xzf splunk_offline_docs.tgz -C $SPLUNK_HOME/etc/apps/
chown -R splunk:splunk $SPLUNK_HOME/etc/apps/splunk_offline_docs
$SPLUNK_HOME/bin/splunk restart

Updating content

Admins can use the Configuration view:

  • Check now — compare local versions against help.splunk.com
  • Incremental update — fetch missing pages and repair links
  • Full refresh — rebuild navigation and re-sync all products

Daily auto-check is disabled by default for air-gapped deployments. Enable it only when the Splunk server can reach help.splunk.com on a schedule.

REST API

Admin endpoint prefix:

/servicesNS/nobody/splunk_offline_docs/offline_docs/

Actions: status, check, update (POST), settings (GET/POST).

Requirements

  • Splunk Enterprise 9.x+ (tested on 9.3.1)
  • ~700 MB–1 GB disk for the documentation bundle
  • Outbound HTTPS only needed for checks/updates (not for day-to-day browsing)

Building a new bundle

On a connected build host with the project repository:

cd Splunk4Offlinedocs
bash scripts/build_bundle.sh

Transfer artifacts/splunk_offline_docs.tgz to the target Splunk server and install.
